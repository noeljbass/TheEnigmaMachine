How it Worked

A practical guide before you use the simulator

Imagine Enigma as an electric maze. Every letter you type travels through plugboard cables, moving rotors, a reflector, and then back through the rotors before a lamp lights up.

1

Rotors are moving scrambled alphabets

A rotor changes one letter into another using hidden wiring. After each keypress, the right rotor moves. Sometimes it pushes the middle rotor, and the middle rotor can push the left rotor. That movement is why pressing the same letter twice can produce different results.

Try this: in the simulator, keep the same settings and type AAAAA. Notice that the output letters are not all the same.

2

The reflector bounces the signal back

The reflector acts like a mirror for the electrical signal. It sends the signal back through the rotors in reverse. This is why the same setup encrypts and decrypts: if the settings are reset exactly, ciphertext travels the same path backward to reveal the original message.

Try this: encrypt a word, reset to the same starting positions, paste the ciphertext, and run it again.

3

Plugboard pairs swap letters before and after

Plugboard cables connect letters in pairs. If AB is connected, A becomes B and B becomes A before entering the rotors. The swap happens again after the signal comes back out. A letter can only be used in one pair, so AB AC is invalid because A appears twice.

Try this: compare the same message with no plugboard, then with AB CD EF.

4

Daily key sheets kept operators synchronized

German units used shared secret key sheets prepared in advance. Everyone with the correct sheet could look up today’s rotor order, starting positions, and plugboard pairs without asking over the radio. If two operators set the same daily key, their machines matched.

Try this: write down your simulator settings before encrypting. You need the exact same settings to decrypt.

The signal path

Keyboard → Plugboard → Right rotor → Middle rotor → Left rotor → Reflector → Left rotor → Middle rotor → Right rotor → Plugboard → Lamp

JavaScript simulator

How the code mimics the mechanical original

The simulator turns the physical Enigma parts into small JavaScript functions. Rotor wiring becomes lookup strings, rotor windows become number positions from 0 to 25, and every keypress advances the machine before sending the signal through the same path as the original hardware.

1. Rotor wiring is stored like a scrambled alphabet

Each rotor object has a wiring string and a notch. The wiring string tells JavaScript which output letter appears at each input position, just like the hidden wires inside a real rotor. 

const rotors = {
  I: { wiring: "EKMFLGDQVZNTOWYHXUSPAIBRCJ", notch: "Q" },
  II: { wiring: "AJDKSIRUXBLHWTMCQGZNPYFVOE", notch: "E" },
  III: { wiring: "BDFHJLCPRTXVZNYEIWGAKMUSQO", notch: "V" },
};

2. The plugboard becomes a two-way map

A real plugboard cable swaps two letters in both directions. The JavaScript version creates a Map where every letter starts by pointing to itself, then valid pairs overwrite both directions. 

const map = new Map(alphabet.split("").map((letter) => [letter, letter]));

map.set("A", "B");
map.set("B", "A");

3. Rotor stepping happens before the signal moves

Enigma moved its rotors as the key was pressed. The code follows that order by calling stepRotors(state) before converting the letter. The notch checks imitate the moments when one rotor pushes the next rotor forward. 

function stepRotors(state) {
  const middleAtNotch = atNotch(state.names[1], state.positions[1]);
  const rightAtNotch = atNotch(state.names[2], state.positions[2]);

  if (middleAtNotch) {
    state.positions[0] = (state.positions[0] + 1) % 26;
    state.positions[1] = (state.positions[1] + 1) % 26;
  } else if (rightAtNotch) {
    state.positions[1] = (state.positions[1] + 1) % 26;
  }

  state.positions[2] = (state.positions[2] + 1) % 26;
}

4. One letter follows the full electrical loop

The function below is the software version of a lamp lighting up: plugboard, right-to-left rotors, reflector, left-to-right rotors, and plugboard again. Because the reflector sends the signal back through the same machine, the same settings can decrypt the message. 

function encipherLetter(letter, state, plugboard) {
  stepRotors(state);

  let signal = letterToIndex(plugboard.get(letter));
  signal = passThroughRotorForward(signal, state.names[2], state.positions[2]);
  signal = passThroughRotorForward(signal, state.names[1], state.positions[1]);
  signal = passThroughRotorForward(signal, state.names[0], state.positions[0]);
  signal = letterToIndex(reflectors[state.reflector][signal]);
  signal = passThroughRotorBackward(signal, state.names[0], state.positions[0]);
  signal = passThroughRotorBackward(signal, state.names[1], state.positions[1]);
  signal = passThroughRotorBackward(signal, state.names[2], state.positions[2]);

  return plugboard.get(indexToLetter(signal));
}
